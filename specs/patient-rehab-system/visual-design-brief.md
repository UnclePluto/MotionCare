# Web 后台视觉稿 Brief

> 状态：草稿 0.1
> 更新日期：2026-05-06
> 用途：作为 Pencil 产出医院康复系统 Web 后台视觉稿的前置说明
> 关联文档：`prd.md`、`implementation-plan.md`

## 1. 视觉稿目标

本视觉稿用于确认医院 Web 后台第一版的页面结构、信息密度、关键流程和交互状态。它不是品牌官网，也不是患者端小程序稿；第一版视觉重点是医生在后台高效完成研究数据闭环。

视觉稿应覆盖：

- 全局患者管理
- 医疗实验项目管理
- 项目分组和随机分组
- 项目患者详情
- T0/T1/T2 访视填写
- 处方管理和处方历史
- 后台代患者训练录入
- 每日健康数据录入
- CRF 预览、缺失字段和导出

视觉稿暂不覆盖：

- 微信小程序
- 真实游戏界面
- 视频上传和 AI 动作识别
- 智能设备同步配置
- 电子签名和复杂质控流转

## 2. 产品气质

这是医院研究和康复管理后台，视觉风格应安静、清晰、专业、偏工具型。

设计原则：

- 信息密度适中偏高，方便医生扫描和批量处理。
- 页面以表格、表单、状态标签、步骤条、抽屉和弹窗为主。
- 不做营销式大标题、插画式空页面、过重卡片堆叠。
- 页面区块边界清楚，但避免把页面整体做成大量浮动卡片。
- 表单字段要有清晰分组，长表单必须支持分段、锚点或标签页。
- 关键状态必须可见，例如分组待确认、已确认、访视草稿、访视已完成、处方生效中。
- 医生需要知道“当前在全局患者维度，还是项目患者维度”。

推荐组件体系：

- 前端实现目标为 React + TypeScript + Ant Design。
- 视觉稿应优先使用 Ant Design 常见模式。
- 主要组件：Layout、Menu、Table、Form、Tabs、Steps、Descriptions、Tag、Alert、Modal、Drawer、DatePicker、Select、InputNumber、Upload 占位、Progress、Statistic、Timeline。

## 3. 全局信息架构

### 3.1 左侧导航

建议一级导航：

- 总览
- 患者档案
- 研究项目
- 训练记录
- 每日健康数据
- CRF 报告
- 基础配置
- 账号权限

导航说明：

- “患者档案”面向全局患者。
- “研究项目”面向项目、分组、项目患者、访视和处方。
- “训练记录”和“每日健康数据”既可作为全局列表入口，也应从项目患者详情进入。
- “CRF 报告”支持按项目、分组、患者筛选和批量查看。

### 3.2 顶部栏

顶部栏内容：

- 当前系统名称：医院康复研究后台
- 当前登录医生姓名
- 角色标签：超级管理员、管理员、医生
- 帮助入口或文档入口
- 退出登录

### 3.3 面包屑

涉及项目患者页面时必须显示清晰路径：

```text
研究项目 / 认知衰弱数字疗法研究 / 项目患者 / 张三
```

原因：同一个患者可以加入多个项目，医生需要明确当前上下文。

## 5. 页面设计说明

### P01 总览

目标：让医生快速知道当前研究数据进度。

页面结构：

- 顶部统计区：
  - 患者总数
  - 进行中项目数
  - 待确认分组批次数
  - 今日需处理访视数
  - CRF 存在缺失字段人数
- 中部左右布局：
  - 左侧：近期项目列表
  - 右侧：待处理事项
- 下方：
  - 最近训练记录
  - 最近 CRF 导出记录

关键状态：

- 待确认分组用橙色标签。
- CRF 缺失用红色或警告色标签。
- 已完成访视用绿色标签。

视觉注意：

- 总览不要做成大屏驾驶舱，不需要复杂图表。
- 重点是待办和入口。

### P02 患者列表

目标：维护全局患者基础档案。

页面结构：

- 顶部筛选：
  - 姓名
  - 手机号
  - 主治医生
  - 状态
- 操作区：
  - 新建患者
  - 批量导入，第一版可先置灰或不出现
- 表格列：
  - 姓名
  - 性别
  - 年龄
  - 手机号
  - 主治医生
  - 已加入项目数
  - 最近更新时间
  - 状态
  - 操作

操作：

- 查看详情
- 编辑基础档案
- 加入项目，可跳转到项目内流程，第一版也可仅从项目详情加入

空状态：

- 文案：暂无患者档案
- 操作：新建患者

### P03 患者详情

目标：查看全局患者信息和其参与的项目。

页面结构：

- 顶部摘要：
  - 姓名、性别、年龄、手机号、主治医生、状态
  - 编辑按钮
- 标签页：
  - 基础信息
  - 所属项目
  - 每日健康数据
  - 操作记录，第一版可后置

所属项目表格：

- 项目名称
- 分组
- 入组时间
- 分组状态
- 最近访视状态
- 当前处方状态
- CRF 缺失数
- 进入项目患者详情

设计重点：

- 患者详情不是完整 CRF 表单。
- CRF 研究数据应放在项目患者详情和访视里。

### P04 项目列表

目标：管理多个医疗实验项目。

页面结构：

- 顶部筛选：
  - 项目名称
  - 项目状态
  - 创建医生
- 操作区：
  - 新建项目
- 表格列：
  - 项目名称
  - CRF 模板版本
  - 访视计划
  - 分组数量
  - 项目患者数
  - 待确认分组批次
  - CRF 缺失人数
  - 状态
  - 操作

项目状态：

- 草稿
- 进行中
- 已归档

### P05 项目详情

目标：在项目上下文里管理分组、患者、访视和 CRF。

页面结构：

- 顶部项目摘要：
  - 项目名称
  - 状态
  - CRF 模板版本
  - 访视计划：T0/T1/T2
  - 分组比例摘要
- 标签页：
  - 项目概览
  - 分组配置
  - 项目患者
  - 随机分组
  - CRF 报告

项目概览：

- 项目患者数
- 各组人数
- 待确认分组批次数
- T0/T1/T2 完成率
- CRF 缺失情况

分组配置：

- 分组表格：组名、目标比例、当前确认人数、说明、状态
- 新建分组按钮
- 分组比例提示：默认 1:1

项目患者：

- 表格列：
  - 患者姓名
  - 性别
  - 年龄
  - 手机号
  - 当前分组
  - 分组状态
  - T0/T1/T2 状态
  - 当前处方
  - CRF 缺失数
  - 操作

### P06 随机分组批次

目标：从全局患者列表选择患者加入项目，并完成分组确认。

建议做成三步流程：

```text
选择患者 -> 生成分组草案 -> 确认分组
```

步骤一：选择患者

- 左侧或上方筛选全局患者：
  - 姓名
  - 手机号
  - 主治医生
- 表格列：
  - 姓名、性别、年龄、手机号、主治医生
  - 是否已加入当前项目
- 已加入当前项目的患者不可再次选择。

步骤二：分组草案

- 展示分组比例说明。
- 表格列：
  - 患者姓名
  - 系统分配分组
  - 当前分组
  - 调整分组 Select
- 分组草案状态标签：待确认。
- 医生可调整分组，不需要填写原因。

步骤三：确认分组

- 展示确认前摘要：
  - 本批次患者数
  - 各组人数
  - 目标比例和实际比例
- 主按钮：确认分组
- 确认后提示：
  - 分组结果确认后不可修改

确认后页面：

- 状态变为已确认。
- 分组 Select 变为只读。
- 显示确认人和确认时间。

### P07 项目患者详情

目标：项目内患者的工作台。

页面结构：

- 顶部摘要：
  - 患者姓名、性别、年龄、手机号
  - 项目名称
  - 当前分组
  - 分组状态
  - 入组时间
- 右侧状态摘要：
  - T0/T1/T2 状态
  - 当前处方状态
  - 训练记录数
  - CRF 缺失字段数
- 标签页：
  - 访视记录
  - 处方
  - 训练记录
  - 健康数据
  - CRF 预览

访视记录 tab：

- T0/T1/T2 卡片或表格。
- 每个访视展示：
  - 状态：草稿、已完成
  - 访视日期
  - 最后编辑时间
  - 编辑入口

### P08 访视表单

目标：医生逐次填写 T0/T1/T2 CRF 相关表单。

布局建议：

- 左侧锚点导航或分段目录。
- 右侧主表单。
- 顶部固定操作条：
  - 保存草稿
  - 标记已完成
  - 返回项目患者详情

T0 表单分段：

- 访视信息
- 知情同意
- 纳入标准
- 排除标准
- 筛选结论
- 人口学资料
- 手术史与过敏史
- 既往病史与家族史
- 行为习惯史
- 基线服药情况
- 身体机能评估
- MoCA 认知评估

T1/T2 表单分段：

- 访视信息
- 干预执行与依从性
- 身体机能评估
- MoCA 认知评估
- 满意度调查
- 合并用药变化
- 不良事件与安全性
- T2 额外包含研究完成/退出和 CRF 质控核查表字段

设计重点：

- 长表单不要一次铺满。
- 每个分段有完成状态提示。
- 字段缺失不阻止保存或完成。
- “已完成”不是锁定，第一版仍可编辑。

### P09 处方管理

目标：医生为项目患者开设版本化处方，并查看历史。

页面结构：

- 顶部提示：
  - 处方是项目患者维度下的版本化训练计划。
  - 训练录入只使用当前生效处方。
- 当前处方卡片：
  - 版本号
  - 状态：草稿、生效中、待生效、已归档、已终止
  - 开设医生
  - 开设时间
  - 生效时间
  - 动作数量
  - 操作：调整处方、终止处方
- 处方动作表格：
  - 动作名称
  - 内部类型
  - 动作类型
  - 频次
  - 时长
  - 组数
  - 难度
  - 注意事项
- 历史版本 Timeline：
  - V1、V2、V3
  - 状态
  - 生效时间
  - 查看快照

新建/调整处方抽屉：

- 选择动作库动作
- 展示动作快照预览
- 配置频次、时长、组数、难度、注意事项
- 生效时间：
  - 默认立即生效
  - 可选择未来生效时间

关键提醒：

- 调整处方会生成新版本，不覆盖旧处方。
- 旧处方只可查看，不支持基于旧处方补录训练。

### P10 后台训练录入

目标：在没有小程序时，医生基于当前处方模拟患者训练数据。

页面入口：

- 项目患者详情的训练记录 tab
- 全局训练记录列表

录入表单结构：

- 项目患者，只读或选择
- 当前生效处方，只读
- 当前处方动作 Select
- 训练日期
- 完成状态
- 实际时长
- 得分
- 备注

根据动作内部类型显示附加字段：

视频类：

- 动作完成情况
- 医生评分
- 动作问题描述

游戏互动类：

- 游戏得分
- 正确率
- 错误次数
- 完成时长
- 难度

运动类：

- 完成组数
- 运动时长
- 主观难度
- 不适情况

异常状态：

- 当前无生效处方：展示 Alert，禁用录入表单，引导先开处方。
- 选择非当前处方动作：前端不提供入口，后端仍应校验。

### P11 每日健康数据

目标：医生手动录入每日健康数据，后续可被 CRF 和统计使用。

页面结构：

- 顶部筛选：
  - 患者
  - 日期范围
- 表格列：
  - 日期
  - 患者
  - 步数
  - 运动时长
  - 平均心率
  - 最高心率
  - 最低心率
  - 睡眠时长
  - 备注
  - 操作

新增/编辑表单：

- 日期
- 步数
- 运动时长
- 平均心率
- 最高心率
- 最低心率
- 睡眠时长
- 备注

规则提示：

- 同一患者同一天只能有一条健康数据。
- 重复录入时进入编辑原记录。

### P12 CRF 预览和导出

目标：生成项目患者 CRF 预览，提示缺失字段，并导出 DOCX/PDF。

页面结构：

- 顶部选择区：
  - 项目
  - 分组
  - 患者
- 项目患者摘要：
  - 患者姓名
  - 项目
  - 分组
  - T0/T1/T2 状态
  - 当前处方状态
- 缺失字段面板：
  - 缺失字段数量
  - 按模块分组：T0、T1、T2、处方、训练、健康数据
  - 点击缺失项跳转到对应编辑入口
- CRF 预览区：
  - 按 CRF 模板章节展示
  - 第一版可以用结构化预览，不要求完全 Word 版式
- 导出区：
  - 导出 DOCX
  - 导出 PDF
  - 导出记录列表

关键文案：

- “允许带缺失字段导出，导出文件中缺失字段留空。”
- “导出记录会保存导出人、导出时间、模板版本和缺失字段清单。”

## 6. 关键流程图建议

Pencil 视觉稿中建议补充 3 个流程图页面。

### F01 项目入组和随机分组流程

```text
创建项目 -> 创建项目分组 -> 从全局患者列表选择患者 -> 生成随机分组草案 -> 确认前调整 -> 确认分组 -> 生成访视入口
```

### F02 项目患者研究数据流程

```text
项目患者详情 -> 填写访视 -> 开设处方 -> 后台录入训练 -> 录入健康日数据 -> CRF 预览 -> 导出 DOCX/PDF
```

### F03 处方版本流程

```text
处方 V1 生效 -> 医生调整 -> 生成处方 V2 -> V2 生效 -> V1 归档 -> 训练录入只使用当前处方
```

## 7. 组件和状态规范

### 7.1 标签颜色建议

| 状态 | 建议颜色 |
| --- | --- |
| 草稿 | 默认灰色 |
| 已完成 | 绿色 |
| 待确认 | 橙色 |
| 已确认 | 蓝色或绿色 |
| 生效中 | 绿色 |
| 待生效 | 蓝色 |
| 已归档 | 灰色 |
| 已终止 | 红色 |
| 缺失字段 | 红色或橙色 |

### 7.2 常用操作模式

- 新建简单对象：Modal
- 编辑长表单：独立页面
- 编辑处方：Drawer 或独立页面，建议先用 Drawer 展示交互思路
- 查看历史：Timeline
- 表格行详情：跳转详情页
- CRF 缺失项：可点击跳转来源页面

### 7.3 空状态文案

| 场景 | 文案 | 主操作 |
| --- | --- | --- |
| 无患者 | 暂无患者档案 | 新建患者 |
| 项目无分组 | 当前项目还没有分组，创建分组后才能随机分组 | 新建分组 |
| 无生效处方 | 当前患者没有生效处方，无法录入训练 | 新建处方 |
| 无健康数据 | 暂无每日健康数据 | 新增健康数据 |
| CRF 无缺失 | 当前 CRF 暂无缺失字段 | 导出 CRF |

视觉稿 review 时重点检查：

- 是否清楚区分全局患者和项目患者。
