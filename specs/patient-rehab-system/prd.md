# 医院病人康复系统 PRD

> 状态：草稿
> 版本：0.2
> 更新日期：2026-05-06
> 当前范围：医院 Web 后台研究数据闭环版
> 文档语言：除专业名词外全部使用中文

## 1. 文档目的

本 PRD 定义医院病人康复系统第一版的产品边界、核心对象、业务流程、功能需求、数据规则、CRF 输出要求和验收标准。

本项目希望采用 spec 方式管理需求。后续研发计划、数据模型、接口契约、视觉稿和验收测试都应基于本 PRD 继续拆分。

本版本基于以下材料整理：

- Notion 页面《病人康复系统》
- Notion 页面《游戏列表》
- 本地模板 `docs/认知衰弱数字疗法研究_CRF表_修订稿.docx`
- 本地接口资料 `docs/OpenAPI接口说明_V1.6.9.docx`
- 本地协议资料 `docs/通信协议.pdf`
- 2026-05-06 需求讨论结论

## 2. 第一版产品定位

第一版定位为 **医院 Web 后台研究数据闭环版**。

核心目标不是先做微信小程序，也不是先接真实智能设备，而是先让医院医生可以在 Web 后台完成：

- 患者基础档案管理
- 医疗实验项目管理
- 项目分组管理
- 患者加入项目和随机分组
- T0/T1/T2 访视数据录入
- 医生开设处方
- 后台代患者录入训练数据
- 手动录入每日健康数据
- CRF Web 预览
- 按现有 Word 模板导出 DOCX/PDF

微信小程序、真实设备 OpenAPI 同步、真实游戏、视频上传、AI 动作识别均后置。第一版通过 Web 后台提供简化的“代患者录入”能力，用于测试、人工补录和 mock 患者训练数据。

## 3. 产品目标

### 3.1 业务目标

- 医生可以维护全局患者基础档案。
- 医生可以创建多个医疗实验项目。
- 医生可以在每个项目内创建多个分组，并配置分组比例。
- 医生可以从全局患者列表中选择患者加入项目。
- 系统可以按项目分组比例生成随机分组草案。
- 医生可以在分组确认前调整分组草案，确认后分组结果锁定。
- 医生可以按项目访视计划填写患者 T0/T1/T2 访视数据。
- 医生可以为项目患者开设版本化处方。
- 医生可以基于当前生效处方，代患者录入视频类、游戏类、运动类训练记录。
- 医生可以按日期手动录入每日健康数据。
- 系统可以汇总患者、项目、分组、访视、处方、训练、健康数据生成 CRF。

### 3.2 研究目标

- 支持认知衰弱数字疗法研究相关数据采集。
- 支持 T0 筛选/入组、T1 干预 12 周、T2 干预后 36 周随访。
- 支持 CRF 模板中的人口学资料、知情同意、纳排标准、病史、行为习惯、基线用药、身体机能评估、MoCA、干预依从性、满意度、用药变化、不良事件、研究完成/退出等数据。
- 支持 CRF 缺失字段提示，但第一版允许带缺失导出，导出文件中缺失字段留空。

## 4. 本期范围

### 4.1 本期包含

- Web 后台账号与权限
- 全局患者基础档案
- 医疗实验项目
- 项目分组和分组比例
- 患者加入项目
- 项目内随机分组草案、分组调整、分组确认
- 项目访视计划和项目患者访视入口
- T0/T1/T2 访视表单
- 处方动作库
- 项目患者处方管理
- 处方版本、处方动作快照、处方历史
- Web 后台代患者训练录入
- 每日健康数据手动录入
- CRF Web 预览、缺失字段提示、DOCX/PDF 导出
- 基础统计和列表筛选

### 4.2 本期不包含

- 微信小程序患者端
- 真实游戏交互
- 视频上传
- AI 动作识别评分
- 智能设备 OpenAPI 自动同步
- 设备 TCP 协议直连
- 电子签名
- 复杂质控流转
- 自动临床诊断或自动研究结论
- 统计显著性分析

## 5. 用户角色

| 角色 | 使用端 | 说明 | 第一版权限 |
| --- | --- | --- | --- |
| 超级管理员 | Web 后台 | 系统最高权限账号 | 管理医生、项目、患者、配置、CRF 模板 |
| 管理员 | Web 后台 | 具备管理权限的医生或运营账号 | 管理医生、患者、项目、配置 |
| 医生 | Web 后台 | 主要使用者 | 建档、建项目、分组、访视、处方、训练录入、健康数据、CRF |

第一版不单独设计患者端账号能力。患者手机号可作为未来小程序账号字段预留。

研究评估员、质控员是否作为独立角色后置确认。第一版默认由医生完成访视填写和 CRF 导出。

## 6. 核心对象模型

### 6.1 全局患者

患者是全局基础档案，不依赖项目存在。

“患者池”只是全局患者列表的口语表达，不是系统中的独立业务对象，也不存在多个患者池。

核心字段：

- 患者 ID
- 姓名
- 性别
- 年龄
- 出生日期
- 手机号
- 主治医生
- 症状或备注
- 创建人
- 创建时间
- 最后编辑人
- 最后编辑时间
- 状态

### 6.2 医疗实验项目

项目用于承载研究或康复管理。

核心字段：

- 项目 ID
- 项目名称
- 项目描述
- CRF 模板版本
- 访视计划
- 分组策略
- 创建医生
- 创建时间
- 项目状态

项目创建后，医生可以在项目内创建多个分组。一个系统中可以存在多个项目。

### 6.3 项目分组

分组属于项目。

核心字段：

- 分组 ID
- 项目 ID
- 分组名称
- 分组说明
- 目标比例
- 排序

默认分组比例为 1:1。项目可以配置多个分组及其比例。

### 6.4 项目患者

项目患者表示“某个全局患者加入某个项目”的关系。

核心字段：

- 项目患者 ID
- 项目 ID
- 患者 ID
- 当前分组 ID
- 入组时间
- 分组状态
- 分组确认时间
- 分组确认人

规则：

- 同一患者可以加入多个项目。
- 同一患者在同一项目内只能存在一条项目患者记录。
- 同一患者在同一项目内只能属于一个分组。
- 项目患者的访视、处方、训练记录、CRF 都基于项目患者维度维护。

### 6.5 分组批次

为了支持“已有项目后继续加入新患者”，随机分组建议按批次处理。

一个分组批次表示医生在某次操作中选择一批患者加入项目，并生成随机分组草案。

核心字段：

- 分组批次 ID
- 项目 ID
- 批次状态：待确认、已确认
- 创建人
- 创建时间
- 确认人
- 确认时间

规则：

- 医生从全局患者列表选择患者加入项目。
- 系统根据项目分组比例生成该批次的随机分组草案。
- 批次确认前，医生可以人工调整该批次内患者所属分组，不需要填写调整原因。
- 批次确认后，该批次内患者的分组结果锁定，不允许再修改。
- 后续新患者可以通过新的分组批次加入同一项目。

### 6.6 访视计划与访视记录

项目绑定访视计划。第一版默认访视节点：

- T0 筛选/入组
- T1 干预 12 周
- T2 干预后 36 周随访

患者加入项目后，系统按项目访视计划生成项目患者的访视入口。

访视状态：

- 草稿
- 已完成

第一版不做待确认、质控、锁定流程。医生填写完成后标记已完成。已完成访视仍允许授权医生编辑，系统记录最后编辑人和最后编辑时间。

### 6.7 处方动作库

处方动作是处方中的可选训练项。

核心字段：

- 动作 ID
- 动作名称
- 训练类型：运动训练、认知训练、高温/雨季适配训练
- 内部类型：视频类、游戏互动类、运动类
- 动作类型：有氧训练、平衡训练、抗阻训练、记忆力、执行力、视空间等
- 执行描述
- 动作要点
- 建议频次
- 建议时长
- 建议组数
- 默认难度
- 状态

第一版动作库用于医生开处方和后台模拟训练录入，不实现真实视频、真实游戏或 AI 评分。

### 6.8 处方

处方属于项目患者，是医生为患者开设的版本化训练计划。它表面上是运动/训练动作列表，但产品上代表某一时间点开始生效的训练依据。

核心字段：

- 处方 ID
- 项目患者 ID
- 版本号
- 开设医生
- 开设时间
- 生效时间
- 状态：草稿、生效中、待生效、已归档、已终止
- 备注

规则：

- 一个项目患者同一时间只有一个生效处方。
- 处方第一版不设置计划周期，也不设置自动结束日期。
- 处方生效后持续有效，直到医生调整生成新版本，或医生手动终止。
- 医生调整处方时，系统基于当前处方生成新版本。
- 新版本默认立即生效，也允许医生选择未来生效时间。
- 新版本生效后，旧版本自动归档。
- 医生可以查看患者在项目内的处方历史。

### 6.9 处方动作快照

处方中的动作采用“引用动作库 + 保存关键快照”。

处方动作字段：

- 处方动作 ID
- 处方 ID
- 动作库动作 ID
- 动作名称快照
- 训练类型快照
- 内部类型快照
- 动作类型快照
- 执行描述快照
- 频次
- 时长
- 组数
- 难度
- 注意事项
- 排序

这样做可以保证动作库后续修改不会影响旧处方、旧训练记录和 CRF 解释。

### 6.10 训练记录

训练记录属于项目患者，并基于当前生效处方录入。

第一版规则：

- Web 后台代患者录入训练记录。
- 训练记录只能基于当前生效处方下的处方动作录入。
- 第一版不支持按旧处方版本补录历史训练。
- 如果当前没有生效处方，不允许录入训练记录。

训练类型：

- 视频类：用表单模拟，不上传真实视频。
- 游戏互动类：用表单模拟，不进入真实游戏。
- 运动类：用表单模拟，不依赖设备。

### 6.11 每日健康数据

第一版不接真实设备，由医生按日期手动录入每日健康数据。

字段：

- 患者 ID
- 日期
- 步数
- 运动时长
- 平均心率
- 最高心率
- 最低心率
- 睡眠时长
- 备注

规则：

- 每个患者每天最多一条健康数据。
- 如果同一天已有记录，编辑原记录，不创建重复记录。
- 健康日数据是患者维度数据。CRF 或项目统计使用时，按项目时间范围或访视时间范围汇总。

### 6.12 CRF 报告

CRF 报告基于项目患者生成。

数据来源：

- 全局患者基础档案
- 项目和分组
- 项目患者访视记录
- 项目患者处方历史
- 项目患者训练记录
- 患者每日健康数据

第一版支持：

- Web 预览
- 缺失字段提示
- DOCX 导出
- PDF 导出
- 导出记录

缺失字段策略：

- Web 预览必须展示缺失字段清单。
- 允许带缺失导出。
- 导出文件中缺失字段留空。
- 第一版不要求导出前必须把缺失字段标记为 88 或 99。

## 7. 核心业务流程

### 7.1 患者管理流程

1. 医生进入患者列表。
2. 医生创建患者基础档案。
3. 患者保存到全局患者列表。
4. 患者可以暂时不加入任何项目。
5. 后续医生可从项目内选择该患者加入项目。

### 7.2 项目和分组管理流程

1. 医生创建医疗实验项目。
2. 医生选择或配置 CRF 模板版本。
3. 医生选择或配置访视计划，第一版默认 T0/T1/T2。
4. 医生创建项目分组。
5. 医生为分组配置比例，默认 1:1。
6. 项目可进入患者入组和随机分组流程。

### 7.3 患者加入项目和随机分组流程

1. 医生进入项目详情。
2. 医生查看可加入项目的全局患者列表。
3. 医生选择一个或多个患者加入项目。
4. 系统创建分组批次。
5. 系统按项目分组比例生成随机分组草案。
6. 医生在确认前可以人工调整草案中的患者分组。
7. 医生确认分组。
8. 系统锁定该批次患者的分组结果。
9. 系统为已确认加入项目的患者生成访视入口。

### 7.4 访视填写流程

1. 医生进入项目患者详情。
2. 医生选择 T0、T1 或 T2 访视。
3. 医生填写访视数据，可保存草稿。
4. 医生填写完成后标记已完成。
5. CRF 生成时读取最新访视数据。

### 7.5 处方管理流程

1. 医生进入项目患者详情。
2. 医生创建处方草稿。
3. 医生从动作库选择处方动作。
4. 系统保存动作库引用和动作快照。
5. 医生保存并生效处方。
6. 如患者已有生效处方，旧处方在新处方生效后归档。
7. 医生可查看处方历史。
8. 医生可终止当前生效处方。

### 7.6 后台代患者训练录入流程

1. 医生进入项目患者详情或训练记录页。
2. 系统读取项目患者当前生效处方。
3. 医生选择当前处方中的处方动作。
4. 系统根据处方动作内部类型展示对应模拟表单。
5. 医生填写训练结果。
6. 系统保存训练记录，并绑定当前处方和处方动作。

### 7.7 健康日数据录入流程

1. 医生进入患者健康日数据页。
2. 医生选择日期。
3. 医生录入步数、运动时长、平均心率、最高心率、最低心率、睡眠时长、备注。
4. 如果该患者该日期已有记录，系统进入编辑模式。
5. 数据保存后可用于项目统计和 CRF 汇总。

### 7.8 CRF 生成流程

1. 医生选择项目患者。
2. 系统聚合患者基础档案、项目、分组、访视、处方、训练、健康数据。
3. 系统按 CRF 模板生成 Web 预览。
4. 系统展示缺失字段清单。
5. 医生可直接导出 DOCX/PDF。
6. 导出时缺失字段留空。
7. 系统记录导出人、导出时间、模板版本和缺失字段清单。

## 8. 功能需求

### 8.1 账号权限

- 支持超级管理员、管理员、医生。
- 支持医生账号新增、编辑、停用。
- 医生手机号唯一。
- 停用医生不可登录，但历史数据保留。

### 8.2 患者列表和患者详情

患者列表支持：

- 新建患者
- 搜索姓名、手机号
- 筛选主治医生
- 查看患者已加入项目
- 进入患者详情

患者详情支持：

- 基础档案编辑
- 所属项目列表
- 健康日数据入口
- 从患者详情跳转到某个项目患者详情

### 8.3 项目管理

项目列表支持：

- 新建项目
- 查看项目状态
- 查看患者数量
- 查看分组数量
- 查看 CRF 完整情况概要

项目详情支持：

- 项目基础信息
- CRF 模板版本
- 访视计划
- 分组管理
- 项目患者列表
- 随机分组入口
- CRF 报告入口

### 8.4 分组管理

- 医生可在项目内创建多个分组。
- 每个分组可配置目标比例。
- 默认比例为 1:1。
- 分组用于项目内随机分组。
- 已确认分组结果引用的分组不建议删除；可停用或禁止删除。

### 8.5 随机分组

- 医生从全局患者列表选择患者加入项目。
- 系统生成随机分组草案。
- 随机目标是尽量靠近项目分组比例。
- 分组草案确认前可手动调整。
- 手动调整无需填写原因。
- 分组确认后不可修改。
- 如果项目没有分组，系统提示先创建分组。

### 8.6 访视表单

T0 访视包含：

- 访视信息
- 知情同意
- 纳入标准
- 排除标准
- 筛选结论
- 人口学资料
- 手术史与过敏史
- 既往病史与家族史
- 行为习惯史
- 基线服药情况
- 身体机能评估
- MoCA 认知评估

T1 访视包含：

- 访视信息
- 干预执行与依从性记录
- 身体机能评估
- MoCA 认知评估
- 数字疗法平台满意度调查
- 合并用药变化
- 不良事件与安全性记录

T2 访视包含：

- 访视信息
- 干预执行与依从性记录
- 身体机能评估
- MoCA 认知评估
- 数字疗法平台满意度调查
- 合并用药变化
- 不良事件与安全性记录
- 研究完成/退出记录
- CRF 质控核查表字段，仅作为模板字段录入，不引入质控流程

第一版访视状态只区分草稿和已完成。

### 8.7 处方动作库

- 支持维护视频类、游戏互动类、运动类动作。
- 动作库仅用于开处方和模拟训练录入。
- 第一版不实现真实视频、真实游戏和 AI 评分。
- 已被处方引用的动作不应物理删除。

### 8.8 处方管理

- 处方属于项目患者。
- 处方是版本化训练计划。
- 一个项目患者同一时间只有一个生效处方。
- 处方生效后持续有效，直到调整或终止。
- 第一版不设置处方计划周期和自动结束日期。
- 医生调整处方会生成新版本。
- 新版本默认立即生效，也可选择未来生效时间。
- 处方动作保存动作库引用和关键快照。
- 医生可查看处方历史。

### 8.9 后台训练录入

通用字段：

- 项目患者
- 当前处方
- 当前处方动作
- 训练日期
- 完成状态
- 实际时长
- 得分
- 备注

视频类模拟字段：

- 动作完成情况
- 医生评分
- 动作问题描述
- 备注

游戏互动类模拟字段：

- 游戏得分
- 正确率
- 错误次数
- 完成时长
- 难度
- 备注

运动类模拟字段：

- 完成组数
- 运动时长
- 主观难度
- 不适情况
- 备注

规则：

- 只能基于当前生效处方录入。
- 不支持按旧处方补录历史训练。
- 当前无生效处方时，不允许录入训练。

### 8.10 健康日数据

- 第一版手动录入。
- 不接真实设备。
- 不做 OpenAPI 同步。
- 字段为日期、步数、运动时长、平均心率、最高心率、最低心率、睡眠时长、备注。
- 同一患者同一日期唯一。

### 8.11 CRF 预览和导出

- CRF 基于项目患者生成。
- 支持 Web 预览。
- 支持缺失字段提示。
- 支持按现有 Word 模板生成 DOCX。
- 支持导出 PDF。
- 缺失字段允许留空导出。
- 记录导出人、导出时间、模板版本、缺失字段清单。

## 10. 边界规则和异常处理

### 10.1 项目和分组

- 项目没有分组时，不允许执行随机分组。
- 分组比例未配置时，默认等比例。
- 分组确认前可以调整。
- 分组确认后不可修改。
- 新患者后续加入已存在项目时，通过新的分组批次完成随机分组和确认。

### 10.2 访视

- 访视可保存草稿。
- 访视可标记已完成。
- 第一版不做确认、质控、锁定。
- 字段缺失不阻止保存或完成。

### 10.3 处方

- 当前无生效处方时，不允许训练录入。
- 调整处方会生成新版本，不覆盖旧版本。
- 旧处方版本用于历史查看，不支持基于旧处方补录训练。
- 动作库修改不影响已生成处方动作快照。

### 10.4 健康日数据

- 同一患者同一日期只能有一条记录。
- 重复录入时进入编辑原记录。
- 健康日数据不绑定单一项目，CRF 使用时按项目或访视时间范围汇总。

### 10.5 CRF

- 缺失字段不阻止预览和导出。
- Web 预览必须提示缺失字段。
- DOCX/PDF 导出时缺失字段留空。
- 导出记录必须保留。

## 11. 第一版验收标准

- 可以创建、编辑、停用医生账号。
- 可以创建和编辑全局患者基础档案。
- 可以创建多个医疗实验项目。
- 可以为项目配置 CRF 模板版本和访视计划。
- 可以在项目内创建多个分组并配置比例。
- 可以从全局患者列表选择患者加入项目。
- 可以生成随机分组草案。
- 可以在确认前人工调整分组草案。
- 确认分组后，项目患者分组不可修改。
- 新患者后续加入项目时，可以通过新的分组批次完成随机分组。
- 项目患者生成 T0/T1/T2 访视入口。
- 医生可以填写访视草稿并标记完成。
- 医生可以维护处方动作库。
- 医生可以为项目患者创建处方。
- 处方调整生成新版本，旧版本归档。
- 处方动作保存动作库引用和关键快照。
- 医生可以查看处方历史。
- 医生可以基于当前生效处方录入视频类、游戏类、运动类训练记录。
- 当前无生效处方时不能录入训练。
- 医生可以按日期录入健康日数据。
- 同一患者同一日期健康数据不重复创建。
- 系统可以生成 CRF Web 预览。
- CRF Web 预览可以展示缺失字段。
- 系统可以导出 DOCX/PDF。
- 缺失字段允许留空导出。
