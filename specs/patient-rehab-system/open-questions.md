# 病人康复系统待讨论问题

> 状态：草稿
> 版本：0.2
> 更新日期：2026-05-06

## 已确认决策

| 编号 | 决策 | 结论 |
| --- | --- | --- |
| D001 | 第一版范围 | 采用医院 Web 后台研究数据闭环版 |
| D002 | 小程序端 | 第一版后置，不作为本期必做 |
| D003 | 真实设备同步 | 第一版后置，不接 OpenAPI，不接 TCP 协议 |
| D004 | 患者模型 | 患者是全局基础档案，不依赖项目存在 |
| D005 | 患者池概念 | “患者池”只是全局患者列表的口语表达，不是业务对象 |
| D006 | 项目模型 | 系统支持多个医疗实验项目 |
| D007 | 项目配置 | 项目绑定 CRF 模板版本和访视计划 |
| D008 | 分组模型 | 分组属于项目，项目可有多个分组 |
| D009 | 分组比例 | 项目内分组配置比例，默认 1:1 |
| D010 | 随机分组 | 患者加入项目后，系统按项目分组比例生成随机分组草案 |
| D011 | 分组调整 | 分组确认前医生可人工调整，无需填写原因 |
| D012 | 分组锁定 | 分组确认后不可修改 |
| D013 | 新患者加入已存在项目 | 通过新的分组批次加入并随机分组 |
| D014 | 访视状态 | 第一版仅支持草稿、已完成 |
| D015 | 访视流程 | 不做医生确认、质控、锁定流转 |
| D016 | 处方定位 | 处方是项目患者维度下的版本化训练计划 |
| D017 | 处方周期 | 第一版不设置计划周期和自动结束日期 |
| D018 | 处方调整 | 调整处方生成新版本，旧版本归档 |
| D019 | 处方生效 | 默认立即生效，也允许选择未来生效时间 |
| D020 | 处方动作 | 引用动作库并保存关键快照 |
| D021 | 训练录入 | Web 后台代患者录入训练数据 |
| D022 | 训练和处方版本 | 第一版只允许基于当前生效处方录入训练 |
| D023 | 历史训练补录 | 第一版不支持基于旧处方补录历史训练 |
| D024 | 训练类型 | 区分视频类、游戏互动类、运动类，但都用表单模拟 |
| D025 | 健康数据 | 第一版按日期手动录入每日健康数据 |
| D026 | 健康日字段 | 日期、步数、运动时长、平均心率、最高心率、最低心率、睡眠时长、备注 |
| D027 | CRF 输出 | 支持 Web 预览、DOCX 导出、PDF 导出 |
| D028 | CRF 缺失策略 | 允许带缺失导出，Web 预览提示缺失，导出文件留空 |
