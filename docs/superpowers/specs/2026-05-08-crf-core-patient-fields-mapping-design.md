## 背景

`specs/patient-rehab-system/prd.md` 定义第一版定位为“医院 Web 后台研究数据闭环版”，其中 **CRF 预览与 DOCX/PDF 导出**是核心交付。当前 PRD 对“患者字段”描述偏粗（模块级），不足以支撑以 CRF 模板为真源的字段级导出与长期演进。

本设计以 `docs/other/认知衰弱数字疗法研究_CRF表_修订稿.docx` 为字段真源，补齐“患者侧固化信息 vs 访视侧增量结果”的边界，并产出可验收的差距清单组织方式与最小映射资产。

## 目标与非目标

### 目标（一期）

- 以 **PRD 第 11 章验收标准**为“完成”的唯一口径，整理可点验的交付清单骨架。
- 以 CRF 模板（docx）为真源，明确字段归属：
  - 患者侧（低频写、高频读、可编辑但属于地坪操作）：人口学、手术/过敏、既往病史/家族史、行为习惯、基线常用药。
  - 访视侧（T0/T1/T2 逐次累积）：量表与检查结果、依从性/满意度、不良事件、合并用药变化、完成/退出。
- 定义一期最小的 **字段级 mapping 资产**（不要求一次性覆盖所有字段，但必须可扩展、可验收）。
- 明确一期版本策略：**A）只保留最新值，不做基线版本化**（编辑覆盖；CRF 导出取最新值）。

### 非目标（一期不做）

- 模板版式像素级一致性的实现路线选择与工程落地（仍属于 PRD 第 13 章待确认项）。
- 复杂质控/锁定/电子签名/完整审计日志。
- 分层随机/区组随机等复杂随机分组算法（一期先满足“按比例尽量靠近”的最小实现与可解释性）。

## 输入与真源

### PRD 口径

- 范围：`specs/patient-rehab-system/prd.md`（第 4、6、8、9、10、11、13 章）
- “完成”判定：以第 11 章验收标准为准

### CRF 模板（docx）提取到的关键模块（字段真源）

以 docx 表格为线索（示例：`#T8` 代表模板内某张表）：

- **封面/识别信息**：受试者编号、姓名缩写、筛选/入组日期等（如 `#T0`、访视信息 `#T3/#T15/#T22`）
- **人口学资料（患者固化）**：年龄、出生日期、性别、婚姻、地址、民族、医保类型、文化水平、教育年限、主要照护者、联系电话、收入、自评健康等（`#T8`）
- **手术史/过敏史（患者固化）**（`#T9`）
- **既往病史/家族史（基础病，患者固化）**：疾病清单 + 家族/个人史 + 确诊时间（`#T10`）
- **行为习惯史（患者固化）**：吸烟/饮酒/运动/久坐等（`#T11`）
- **基线用药（患者固化）**：按用途/类别“是否服用”+ 药物名称（可多选/填写）+ 常用药物数量（`#T12`）
- **随访增量结果（访视侧）**：
  - 身体机能评估（SPPB/TUG/握力/衰弱）：`#T13/#T17/#T24`
  - MoCA 分项与总分：`#T14/#T18/#T25`
  - 依从性：`#T16/#T23`
  - 满意度：`#T19/#T26`
  - 合并用药变化：`#T20/#T27`
  - 不良事件：`#T21/#T28`
  - 完成/退出：`#T29`
  - 质控核查（一期仅录入模板字段，不做流程）：`#T30`

## 一期字段归属设计（混合）

### 1) 归属原则

- **患者侧（固化信息）**：录入后通常长期复用，支持编辑但为低频操作；用于 CRF 预填与快速导出。
- **访视侧（增量结果）**：按 T0/T1/T2 分次产生；允许不断补充与修订；用于纵向随访与 CRF 节点化呈现。

### 2) 推荐的数据边界（概念模型）

> 仅定义边界与责任，不限定具体 ORM/迁移实现细节。

- `Patient`：患者主表（基础识别与快速检索字段）
- `PatientBaseline`：患者基线/固化信息（人口学、手术/过敏、病史家族史、行为习惯、基线用药）
  - 性能：常用筛选字段建议列化可索引；病史/药物明细可拆子表或结构化字段（一期实现以“可导出、可检索”为第一优先）。
- `Visit`：访视记录（`project_patient + timepoint(T0/T1/T2)` 唯一；含访视日期、时间窗、评估者等）
- `VisitAssessments`：量表/检查结果（SPPB/TUG/握力/衰弱、MoCA 分项/总分等）
- `VisitEvents`：依从性/满意度/不良事件/合并用药变化/完成退出等事件型数据

### 3) 一期版本策略（已确认）

- **A）只保留最新值**：`PatientBaseline` 修改会覆盖旧值；CRF 导出永远读取最新值。
- 风险提示：若未来需要“按入组时快照导出/审计”，需引入轻量版本化或“入组快照”机制；一期先不实现。

## CRF mapping 的最小交付资产（P0）

### 1) 资产形式

以 docx 表格为单位建立字段级 mapping（后续可逐字段细化），至少覆盖：

- `#T8 人口学资料` → `PatientBaseline.demographics.*`
- `#T9 手术史/过敏史` → `PatientBaseline.surgery_allergy.*`
- `#T10 既往病史与家族史` → `PatientBaseline.comorbidities.*`（含明细）
- `#T11 行为习惯史` → `PatientBaseline.lifestyle.*`
- `#T12 基线用药` → `PatientBaseline.medications.*`（含明细）
- `#T13/#T14` 等评估结果 → `VisitAssessments.*`
- `#T16/#T19/#T20/#T21/#T29` 等事件 → `VisitEvents.*`

### 2) 缺失字段判定与导出规则（一期）

- Web 预览必须给出缺失字段清单（PRD 第 6.12 / 8.11）。
- 允许带缺失导出；导出中缺失字段留空。
- “缺失”的判定需在 mapping 中明确：空字符串/未选择/不适用（如模板提供“不适用”）分别如何处理。

### 3) 导出版本记录（一期）

- 至少记录：导出人、导出时间、CRF 模板版本号（模板内字段）、缺失字段清单。

## 验收驱动差距清单（建议模板）

对 PRD 第 11 章每条验收项，按以下结构拆解并维护状态：

- **验收条目**：PRD 原文
- **落地项**：页面 / 接口 / 数据结构 / 规则 / CRF mapping / 测试用例
- **完成判定**：操作路径 + 期望结果（可点验）
- **依赖与决策点**：是否依赖 PRD 第 13 章（如模板是否继续调整、版式一致性要求等）
- **风险**：最可能返工的点（字段归属、时间窗规则、计算规则等）

## 一期需要进一步细化/确认（阻塞或高风险）

来自 PRD 第 13 章，且与 CRF 强关联：

- CRF 模板是否会继续调整（影响 mapping 演进与导出一致性）
- DOCX/PDF 版式一致性验收口径（影响导出实现路线与工作量）
- T1/T2 访视时间窗规则（影响 `Visit` 与导出字段“是否在时间窗内”）
- MoCA/SPPB 是否系统自动计算（影响 `VisitAssessments` 字段与一致性校验）

## 结论

一期按“验收驱动”推进，并以 CRF 模板为字段真源，采用“患者固化信息 + 访视增量结果”的混合切分；版本策略先按 A（只保留最新值）实现，以最快速度形成可预览、可导出、可验收的 CRF 资产闭环。
