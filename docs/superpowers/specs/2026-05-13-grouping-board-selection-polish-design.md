# 分组看板选择联动与文案收敛设计

日期：2026-05-13  
范围：MotionCare 管理端项目详情页、分组看板患者选择与临时随机结果联动。  
状态：待用户 review 后进入实现计划。

## 背景

项目详情页已经回归为单一分组看板。验收时发现三个小问题：

- 患者已被本地随机到组内后，如果用户在“全量患者”区域取消勾选，该患者仍留在组内的本次随机结果中。
- “全量患者”区域存在冗余说明文案：`勾选未确认入组患者后点击「随机分组」会生成本次页面内临时结果；确认前不会写入后端。`
- 项目详情页右上角按钮仍显示 `新增分组 / 元数据`，需要收敛为 `新增分组`。

## 目标

- 取消勾选未确认患者时，同步删除该患者的本次随机结果。
- 重新勾选该患者时，只把患者放回待随机选择集合，不自动恢复旧分组。
- 删除“全量患者”卡片内的冗余说明文案。
- 将项目详情页右上角按钮文案改为 `新增分组`。

## 非目标

- 不改变已确认患者的禁选、置灰和解绑逻辑。
- 不改变随机算法、分组比例、确认入组接口或后端语义。
- 不新增自动恢复本次随机结果的能力。
- 不改变 Drawer 标题 `分组配置（元数据）`。
- 不删除顶部工具条中关于“当前随机结果仅保存在本页面”的全局说明。

## 交互设计

`ProjectGroupingBoard` 中患者勾选逻辑收敛到一个明确的选择变更函数，例如：

- `handlePatientSelectionChange(patientId, checked)`

当 `checked=true`：

- 如果患者尚未在 `poolSelected` 中，则加入 `poolSelected`。
- 不修改 `localAssignments`。
- 如果该患者此前取消勾选导致本次随机结果被删除，重新勾选后也不会自动恢复旧结果。
- 用户需要再次点击 `随机分组`，才会重新生成本次随机结果。

当 `checked=false`：

- 从 `poolSelected` 删除该患者。
- 从 `localAssignments` 删除 `patientId` 相同的本次随机结果。
- 若删除后没有任何本次随机结果，`确认分组` 按钮应恢复 disabled 状态。

已确认患者仍由 `confirmedPatientIds` 控制为 disabled，不参与选择变更或随机。

## 文案设计

删除“全量患者”卡片内的段落：

```text
勾选未确认入组患者后点击「随机分组」会生成本次页面内临时结果；确认前不会写入后端。
```

保留工具条中的说明：

```text
当前随机结果仅保存在本页面；刷新或切换项目会丢弃，点击「确认分组」后才正式入组。
```

项目详情页右上角按钮文案：

- 旧：`新增分组 / 元数据`
- 新：`新增分组`

## 测试与验收

前端测试应覆盖：

- 勾选未确认患者并点击 `随机分组` 后，组内出现 `本次随机` 卡片。
- 取消该患者勾选后，该患者对应的 `本次随机` 卡片消失。
- 取消后如果没有其他本次随机患者，`确认分组` 按钮 disabled。
- 删除的“全量患者”说明文案不再出现在页面上。
- 项目详情页出现 `新增分组` 按钮，不再出现 `新增分组 / 元数据`。
- 既有本地随机、确认入组、已确认患者置灰测试继续通过。
